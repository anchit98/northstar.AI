Discovery & Validation

expand_less
S

Situation

Legacy product "Alpha" was seeing a 15% MoM churn rate. Leadership wanted to rewrite the entire codebase, assuming technical debt was the root cause.

T

Task

I needed to validate if a technical rewrite would actually solve the user pain points before committing 6 months of engineering bandwidth.

A

Action

Halted the rewrite. Conducted 20 user interviews and implemented Hotjar. Discovered the issue wasn't performance, but a deeply confusing navigation hierarchy introduced in Q3.

R

Result

Pivoted to a 2-week UI redesign instead of a 6-month rewrite. Reduced churn by 22% in the following quarter and saved an estimated $120k in engineering costs.
